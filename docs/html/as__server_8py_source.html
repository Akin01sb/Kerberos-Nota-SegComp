51 """
52 ***************************************************************************
53 Funcao: carregar_usuarios
54
55 @brief Carrega usuarios cadastrados para validacao pelo AS.
56
57 Descricao:
58 Le `data/usuarios.json` e retorna sua estrutura. O arquivo contem salt,
59 verificador e perfil de cada usuario, mas nao contem senha em texto claro.
60
61 Parametros:
62 Nao recebe parametros explicitos.
63
64 Valor retornado:
65 @return Dicionario com a chave `usuarios`.
66
67 Assertiva de entrada:
68 @pre O caminho de usuarios pode existir ou nao.

71 @post Retorna usuarios cadastrados ou estrutura vazia se o arquivo nao existir.
72
73 Excecoes:
74 @throws json.JSONDecodeError Pode ocorrer se o JSON estiver invalido.
75 @throws OSError Pode propagar erro de leitura do arquivo.
76
77 Observacoes:
78 Esta funcao apoia o AS e tambem a camada de notas para descobrir perfis.

88def obter_dados_usuario(nome_usuario: str) -> dict:
89 """
90 ***************************************************************************
91 Funcao: obter_dados_usuario
92
93 @brief Recupera os dados persistidos de um usuario.
94
95 Descricao:
96 Busca o usuario no arquivo carregado e retorna os campos necessarios ao AS,
97 como salt e verificador. Se o usuario nao existir, a autenticacao inicial
98 deve falhar.
99
100 Parametros:
101 @param nome_usuario Identificador textual do usuario.
102
103 Valor retornado:
104 @return Dicionario com os dados do usuario.
105
106 Assertiva de entrada:
107 @pre O nome do usuario deve ser uma string cadastrada.
108
109 Assertiva de saida:
110 @post Retorna os dados do usuario existente.
111
112 Excecoes:
113 @throws ValueError Quando o usuario nao esta cadastrado.
114 @throws Exception Pode propagar erros de leitura do JSON.
115
116 Observacoes:
117 A funcao nao valida senha; ela apenas localiza os dados usados no desafio.
118 ***************************************************************************
119 """
120 usuarios = carregar_usuarios().get("usuarios", {})
121 if nome_usuario not in usuarios:
122 raise ValueError("Usuario nao encontrado.")
123 return usuarios[nome_usuario]

126def criar_desafio_as(nome_usuario: str) -> dict:
127 """
128 ***************************************************************************
129 Funcao: criar_desafio_as
130
131 @brief Cria os parametros AS-REQ para o cliente responder.
132
133 Descricao:
134 Remove desafios expirados, gera um desafio aleatorio, associa esse desafio
135 ao usuario e retorna salt, numero de iteracoes da KDF e desafio. O cliente
136 usa esses dados para derivar a chave localmente e produzir a prova HMAC.
137
138 Parametros:
139 @param nome_usuario Usuario que iniciou a autenticacao.
140
141 Valor retornado:
142 @return Dicionario com usuario, salt, iteracoes_kdf e desafio.
143
144 Assertiva de entrada:
145 @pre O usuario deve existir em `data/usuarios.json`.
146
147 Assertiva de saida:
148 @post Um desafio de uso unico fica registrado em memoria por ate 60 segundos.
149
150 Excecoes:
151 @throws ValueError Quando o usuario nao esta cadastrado.
152 @throws Exception Pode propagar erros de leitura ou aleatoriedade.
153
154 Observacoes:
155 Esta funcao representa a primeira metade da comunicacao com o AS no fluxo
156 academico: o cliente ainda nao recebe TGT, apenas parametros para provar
157 conhecimento da senha.
158 ***************************************************************************
159 """
160 dados_usuario = obter_dados_usuario(nome_usuario)
161 agora = timestamp_atual()
162
163 with BLOQUEIO_DESAFIOS_AS:
164 expirados = [
165 desafio
166 for desafio, dados in DESAFIOS_AS.items()
167 if dados["timestamp"] < agora - TEMPO_VALIDADE_DESAFIO
168 ]
169 for desafio in expirados:
170 DESAFIOS_AS.pop(desafio, None)
171
172 desafio = secrets.token_hex(32)
173 DESAFIOS_AS[desafio] = {
174 "usuario": nome_usuario,
175 "timestamp": agora,
176 }
177 return {
178 "usuario": nome_usuario,
179 "salt": dados_usuario["salt"],
180 "iteracoes_kdf": ITERACOES_PBKDF2,
181 "desafio": desafio,
182 }

185def _emitir_resposta_as(
186 nome_usuario: str,
187 chave_cliente: bytes,
188 validade_segundos: int
189) -> dict:
190 """
191 ***************************************************************************
192 Funcao: _emitir_resposta_as
193
194 @brief Monta a resposta cifrada do AS apos autenticacao bem-sucedida.
195
196 Descricao:
197 Gera a chave de sessao Cliente-TGS, cria o TGT, cifra o TGT com a chave
198 secreta do TGS e cifra a resposta final com a chave de longo prazo do
199 cliente.
200
201 Parametros:
202 @param nome_usuario Usuario autenticado.
203 @param chave_cliente Chave de longo prazo reproduzida pelo cliente.
204 @param validade_segundos Tempo de validade do TGT.
205
206 Valor retornado:
207 @return Pacote AES-GCM contendo chave Cliente-TGS e TGT.
208
209 Assertiva de entrada:
210 @pre A prova do usuario ja deve ter sido validada.
211 @pre A chave do cliente deve ter tamanho aceito pelo AES-GCM.
212
213 Assertiva de saida:
214 @post Retorna AS-REP cifrado e um TGT transportavel pelo cliente.
215
216 Excecoes:
217 @throws ValueError Quando a validade do TGT for invalida.
218 @throws Exception Pode propagar erros de cifragem ou aleatoriedade.
219
220 Observacoes:
221 O cliente transporta o TGT, mas nao consegue abri-lo porque ele e cifrado
222 com a chave secreta do TGS.
223 ***************************************************************************
224 """
225 chave_sessao_cliente_tgs = gerar_chave_simetrica()
226 chave_sessao_cliente_tgs_base64 = bytes_para_base64(chave_sessao_cliente_tgs)
227
228 tgt = gerar_tgt(
229 nome_usuario=nome_usuario,
230 chave_sessao_cliente_tgs_base64=chave_sessao_cliente_tgs_base64,
231 validade_segundos=validade_segundos,
232 )
233 tgt_criptografado = criptografar_json(CHAVE_SECRETA_TGS, tgt)
234 resposta_para_cliente = {
235 "id_tgs": "tgs",
236 "chave_sessao_cliente_tgs": chave_sessao_cliente_tgs_base64,
237 "tgt": tgt_criptografado,
238 "timestamp_emissao": tgt["timestamp_emissao"],
239 "timestamp_expiracao": tgt["timestamp_expiracao"],
240 "validade_segundos": tgt["validade_segundos"],
241 "nonce_tgt": tgt["nonce"],
242 }
243 return criptografar_json(chave_cliente, resposta_para_cliente)

246def autenticar_no_as_com_prova(
247 nome_usuario: str,
248 desafio: str,
249 prova: str,
250 validade_segundos: int = TEMPO_VALIDADE_TGT
251) -> dict:
252 """
253 ***************************************************************************
254 Funcao: autenticar_no_as_com_prova
255
256 @brief Valida a prova HMAC e emite a resposta do AS.
257
258 Descricao:
259 Consome um desafio previamente criado, verifica se pertence ao usuario,
260 confere expiracao e compara a prova HMAC recebida com a prova esperada. Se
261 a validacao passar, emite a resposta do AS com TGT e chave Cliente-TGS.
262
263 Parametros:
264 @param nome_usuario Usuario que responde ao desafio.
265 @param desafio Desafio emitido por criar_desafio_as.
266 @param prova Prova HMAC-SHA256 codificada em Base64.
267 @param validade_segundos Validade desejada para o TGT.
268
269 Valor retornado:
270 @return Pacote cifrado para o cliente com os dados da autenticacao.
271
272 Assertiva de entrada:
273 @pre O desafio deve existir, estar valido e pertencer ao usuario.
274 @pre A prova deve ter sido calculada com a chave derivada da senha correta.
275
276 Assertiva de saida:
277 @post O desafio e removido da memoria e nao pode ser reutilizado.
278 @post Em caso de sucesso, um TGT e emitido.
279
280 Excecoes:
281 @throws ValueError Para desafio invalido, expirado ou prova incorreta.
282 @throws Exception Pode propagar erros de cifragem e leitura de usuarios.
283
284 Observacoes:
285 Esta funcao substitui o envio direto de senha ao AS. O AS valida posse da
286 senha por desafio e HMAC.
287 ***************************************************************************
288 """
289 with BLOQUEIO_DESAFIOS_AS:
290 dados_desafio = DESAFIOS_AS.pop(desafio, None)
291 if not dados_desafio or dados_desafio["usuario"] != nome_usuario:
292 raise ValueError("Desafio de autenticacao invalido.")
293
294 if dados_desafio["timestamp"] < timestamp_atual() - TEMPO_VALIDADE_DESAFIO:
295 raise ValueError("Desafio de autenticacao expirado.")
296
297 dados_usuario = obter_dados_usuario(nome_usuario)
298 chave_autenticacao = base64_para_bytes(dados_usuario["verificador"])
299 prova_esperada = gerar_prova_as(
300 chave_autenticacao,
301 nome_usuario,
302 desafio,
303 )
304 if not hmac.compare_digest(prova_esperada, prova):
305 raise ValueError("Senha invalida.")
306
307 return _emitir_resposta_as(
308 nome_usuario,
309 chave_autenticacao,
310 validade_segundos,
311 )

314def gerar_tgt(
315 nome_usuario: str,
316 chave_sessao_cliente_tgs_base64: str,
317 validade_segundos: int = TEMPO_VALIDADE_TGT
318) -> dict:
319 """
320 ***************************************************************************
321 Funcao: gerar_tgt
322
323 @brief Cria a estrutura do Ticket Granting Ticket.
324
325 Descricao:
326 Usa a funcao comum de tickets para montar o TGT e acrescenta campos usados
327 pelo AS, como usuario, expiracao e nonce. O TGT sera cifrado pela resposta
328 do AS antes de ser entregue ao cliente.
329
330 Parametros:
331 @param nome_usuario Usuario autenticado.
332 @param chave_sessao_cliente_tgs_base64 Chave Cliente-TGS em Base64.
333 @param validade_segundos Tempo de validade do ticket.
334
335 Valor retornado:
336 @return Dicionario com os campos do TGT em claro antes da cifragem.
337
338 Assertiva de entrada:
339 @pre A validade deve ser positiva.
340 @pre A chave de sessao deve estar codificada em Base64.
341
342 Assertiva de saida:
343 @post Retorna TGT contendo identidade, chave de sessao, validade e nonce.
344
345 Excecoes:
346 @throws ValueError Quando a validade for menor ou igual a zero.
347
348 Observacoes:
349 A funcao apenas monta a estrutura; a protecao criptografica ocorre em
350 _emitir_resposta_as.
351 ***************************************************************************
352 """
353 if validade_segundos <= 0:
354 raise ValueError("Validade do TGT invalida.")
355
356 tgt = criar_tgt(
357 id_cliente=nome_usuario,
358 chave_sessao_cliente_tgs_base64=chave_sessao_cliente_tgs_base64,
359 id_tgs="tgs"
360 )
361
362 # Campos extras documentam validade e nonce antes da cifragem do TGT.
363 tgt["usuario"] = nome_usuario
364 tgt["validade_segundos"] = validade_segundos
365 tgt["timestamp_expiracao"] = tgt["timestamp_emissao"] + validade_segundos
366 tgt["nonce"] = uuid.uuid4().hex
367
368 return tgt
